    Install:

      http://www.goodbye-microsoft.com/

    A szakertoi telepitesben jessie-t is ki lehet valasztani. Ujrainditas utan
    a 'try NTFS5 nog2ldr' uzeneten egy ideig megall a folyamat, de ha kivarod,
    folytatodni fog a telepites.

    Windows legyen az elso a listaban:

      $ mv /etc/grub.d/30_os-prober /etc/grub.d/08_os-prober
      $ update-grub

    Automount:

      $ cat /proc/partitions
      $ ls -l /dev/disk/by-uuid
      # A kettot osszenezve megtudhatod a cuccos particio uuid-jet.
      $ cat >> /etc/fstab
      UUID=123456  /mnt  ntfs-3g  defaults  0  0
      $ chown user /mnt

    Konfig fajlok letrehozasa:

      $ rm -r ~/.config/*
      $ cd /mnt/home
      $ ./_symlink.sh
      $ cp -r _config/* ~/.config

                                     FAQ                                  {{{1
==============================================================================

    Framebuffer: boot kozben fekete kepernyo helyett mukodjon a bootsplash:

      $ echo FRAMEBUFFER=y > /etc/initramfs-tools/conf.d/splash
      $ update-initramfs -u

    Splash helyett szoveges boot:

      # /etc/default/grub: GRUB_CMDLINE_LINUX_DEFAULT="quiet splash" -> ""
      $ update-grub

    Splash valtoztatas:

      $ sudo update-alternatives --config default.plymouth

    Fajl-tarsitasok:

      $ xdg-mime default PROGI.desktop $(grep 'MimeType='              \
        /usr/share/applications/PROGI.desktop | sed -e 's/.*=\|;/ /g')

      Ha ez sem mukodik, akkor dpkg-reconfigure CSOMAG - a beallitofajlokat
      nem fogja piszkalni.

    Elrontottam az MBR-t:

      Fogj egy livecd-t, amin van grub, masold at a sajat grub.cfg fajlodat a
      /boot/grub/grub.cfg helyre, majd:

      $ grub-install --root-directory=/MOUNTOLT_MEGHAJTO /dev/sdAKARMI

      Miutan elindult a sajat rendszered, futtasd a grub-install /dev/sdAKARMI
      parancsot ismet.

    Mount-olt meghajtora nem tudok user-kent irni:

      $ chmod 777 /mnt

    Kozos ubuntu csomagok:

      $ apt-cache show '^[kx]?ubuntu-desktop$' | grep 'Depends' |
      $ sed 's/Depends: \|, \?/\n/g' | sort | uniq -c | sort -n -k 1,2

    Nem mukodik a net:

      192.168.1.1 / advanced / adsl / auto -> adsl2

    A hang az xfce indulasakkor el van nemitva:

      $ install alsa-utils, unmute

    Vlc-ben nehany filmnek nincs hangja (ac3 kodek)

      alsamixer: S/PDIF a hibas - vagy a vlc-ben tiltsd le, vagy allitsd be.

    Java programokhoz JRE:

      $ openjdk-7-jre

    xorg.config - hova lett, hogy tudom beallitani?

      X :1 -configure  ->  ~/xorg.conf.new

      Ezt modositsd es masold az /etc/X11/xorg.conf fajlba.
      A :1 a display szama - valszeg hibat fog kiirni, de ettol meg
      legeneralja a xorg.conf-ot.

    Visszavaltas Nvidia-rol Nouveau-ra:

      - tavolitsd el az osszes Nvidia csomagot,
      - echo 'nouveau' >> /etc/modules
      - xorg.config-ot generald ujra es csak a nouveau cuccokat hagyd benne, a
        framebuffer, meg a tobbi mehet a levesbe.

    bash_history nem mukodik:

      Valszeg root a tulajdonos - torold, utanna mukodni fog.

    compiz, opengl - Nem mukodik, csak egy fekete kepernyot latok.

      Nem tudom miert, de az Nvidia GeForce FX5200 keptelen normalisan kezelni
      az opengl-t, ha minden igaz. Legalabbis mikor a compiz-config opengl
      resezet engedelyezem, blackdeath lesz. Probaltam a noveau driverrel es a
      gyari 173-assal is, de egyikkel sem mukodott. (az nvidia-glx megszunt,
      ugy tunik, hogy a glx modul viszont alapbol telepitve van) Bovebb infot
      ad a glxinfo.
